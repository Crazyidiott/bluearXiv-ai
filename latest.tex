Auto-FL-Research (AFR) proposes a constrained coding-agent workflow that lets agents propose and implement candidate training algorithms (e.g., server aggregation, client updates) under fixed task profiles, with each campaign recording scores, runtime, and failure status. Evaluated on five healthcare FLamby tasks and six LEAF profiles, AFR showed gains on most tasks, but also exposed seed-sensitive and search-selected failure cases; same-budget controls revealed that some improvements came from genuine FL-recipe changes, while others were fixed-surface tuning effects or single-run artifacts.
